## 기술 스택 및 시스템 아키텍처
이 문서는 프로젝트에서 사용되는 기술 스택과 시스템 아키텍처 설계에 대해 설명합니다. (2024-08-25 기준)

### 1. 기술 스택

- **프로그래밍 언어**: Python 3.11.9
- **백엔드 프레임워크**: FastAPI
- **프론트엔드 프레임워크**: React.js
- **데이터베이스**: MySQL
- **API 개발**: FastAPI
- **데이터 크롤링 및 처리**: BeautifulSoup, Scrapy, Pandas (필요 시)
- **서버 및 배포**: Nginx (리버스 프록시), Uvicorn (WSGI 서버), Docker (컨테이너화), Oracle Cloud (클라우드 배포)
- **데이터 시각화**: Plotly, D3.js (프론트엔드에서 시각화)
- **사용자 인증 및 보안**: OAuth2 및 JWT (JSON Web Token)
- **검색 및 인덱싱**: SQLAlchemy 또는 Tortoise ORM과 Elasticsearch 통합

### 2. 시스템 아키텍처 설계

#### 백엔드
- FastAPI 서버를 사용해 주요 비즈니스 로직을 처리하고, MySQL과 연동해 데이터베이스를 관리.
- SQLAlchemy 또는 Tortoise ORM을 사용해 데이터베이스와 상호작용하며, RESTful API를 FastAPI 내장 기능으로 구축.

#### 프론트엔드
- React.js를 사용해 사용자 인터페이스를 구성하며, 비동기적으로 FastAPI API로부터 데이터를 받아와 화면에 렌더링.
- Axios 또는 Fetch API를 사용해 FastAPI에서 제공하는 REST API와 통신.
- 다양한 데이터를 시각화하기 위해 Plotly나 D3.js를 사용.

#### 데이터베이스
- **MySQL**:
  - 축구 선수, 팀, 경기, 리그 정보 등의 데이터를 저장.
  - SQLAlchemy 또는 Tortoise ORM을 통해 데이터베이스와 상호작용하며, 복잡한 쿼리도 쉽게 처리.

#### 데이터 크롤링 및 처리 (선택 사항)
- **Scrapy & BeautifulSoup**:
  - 축구 관련 주요 사이트에서 데이터를 수집하고, 이를 MySQL 데이터베이스에 저장.
  - Pandas를 사용해 데이터를 처리하고 정리.

#### 서버 및 배포
- Nginx를 리버스 프록시로 설정하여 HTTP 요청을 Uvicorn으로 전달.
- Uvicorn을 WSGI 서버로 사용해 FastAPI 애플리케이션을 실행.
- Docker를 사용해 애플리케이션을 컨테이너화하여 개발 및 배포 환경을 일관되게 유지.
- Oracle Cloud 인프라에 배포하며, Oracle Cloud 무료 티어를 통해 클라우드 자원을 활용.

#### 사용자 인증 및 보안
- OAuth2 및 JWT를 사용해 사용자 인증을 처리하며, 소셜 로그인 및 토큰 기반 인증을 구현.

#### 검색 기능 (선택)
- Elasticsearch와 FastAPI를 연동해 대용량 데이터에서의 빠른 검색 기능을 제공.
- 팀, 선수, 경기 등의 데이터를 빠르게 검색할 수 있도록 인덱싱을 설정.

### 3. 추가 고려 사항

#### 국내 사용자 경험
- 한국어 UI/UX를 고려해 사용자 친화적인 인터페이스를 설계.
- 모바일 대응을 위한 반응형 웹 디자인을 적용.

#### 데이터베이스 확장성
- 데이터의 양이 증가할 경우를 대비해 샤딩 또는 리플리케이션 설정을 고려.
- 대량의 트래픽을 처리할 수 있도록 서버 확장 및 캐싱 전략을 적용.
